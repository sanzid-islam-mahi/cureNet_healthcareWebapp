# CureNet User Guide

This is a short role-based user guide for demonstrating the platform in the viva.

## 1. Patient Flow

Main features for patients:

- register and verify account
- log in and manage profile
- browse doctors
- book appointments
- view appointment history
- view prescriptions
- manage medication reminders
- review medical history
- view or upload imaging records

Suggested walkthrough:

1. log in as a patient
2. open the dashboard
3. browse a doctor profile
4. book an appointment
5. show appointment history
6. show prescription history and reminders
7. show medical history and imaging

## 2. Doctor Flow

Main features for doctors:

- maintain doctor profile
- manage appointment queue
- view continuity information for patients
- create and edit prescriptions
- review visit-linked imaging
- upload imaging records linked to relevant appointments

Suggested walkthrough:

1. log in as a doctor
2. open doctor dashboard
3. open appointments queue
4. open a patient context view
5. create or review a prescription
6. show reminder-related continuity data
7. show appointment-linked imaging management

## 3. Receptionist Flow

Main features for receptionists:

- access clinic-scoped dashboard
- review clinic appointment queue
- approve or manage operational appointment flow
- view clinic doctor roster
- support appointment-linked imaging workflows where allowed

Suggested walkthrough:

1. log in as a receptionist
2. open receptionist dashboard
3. show clinic queue
4. open clinic doctors view
5. explain that access is limited to the receptionist’s assigned clinic

## 4. Admin Flow

Main features for admins:

- monitor platform dashboard
- review doctor verification
- manage clinics
- manage users
- view analytics and logs

Suggested walkthrough:

1. log in as admin
2. open admin dashboard
3. show clinic management
4. show user management
5. show doctor verification workflow
6. show analytics or audit/log screens

## 6. Demo Tips

- use seeded demo data so dashboards are populated
- log in fresh for each role rather than switching too quickly
- keep the story operational:
  - patient books
  - receptionist handles clinic queue
  - doctor treats and prescribes
  - admin manages the system
